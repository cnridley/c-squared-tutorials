Strings are a collection of characters, numbers and symbols. Strings are how we display text and data, making it one of the most fundamental concepts in JavaScript to understand.
There are three ways of creating a string:
1. With single quotations i.e 

'I am a string'
2. With double quotations i.e 
"I am also a string"
3. Using template literals or backticks i.e 
`You guessed it, I am a string too!`

It does not matter which technique you use, as long as it matched at the beginning and end, otherwise errors can occur
e.g. var string = 'The developer said 'Make sure you use the correct syntax' to the intern.'
console.log(string) => 'The developer said'
This is because JS has ended the string after the second '. To correct this put the quote in double quotations or use template literals.
Each has its advantages and disadvantages which we will go into in more depth further down.

Strings can be stored in variables to be called later on in the code, like the above example.

String Concatenation:

This is the joining of two or more strings using the + symbol.
e.g.

            var book = "Harry Potter"
            var author = "J K Rowling"

            var FaveBook = "My favourite book is " + book + " by" + author.

            console.log(FaveBook);
            My favourite book is Harry Potter by J K Rowling.

Template literals:

Template literals are a relatively new concept to JavaScript. It is simply constructing a string with backticks ``, instead of quotations.
The advantages of this include:
It reduces risk of error by using the incorrect quotations throughout the string.
It provides a multi line support, so \n is not needed. Which reduces the amount of code that needs to be written and makes it look cleaner. e.g. 

        var TemplateString = `This string
                                will print out on multiple
                                lines because we used
                                a template literal`
Another benefit of template literal is string interpolation. This is a way of joining strings together usng ${}. This is quicker, easier and looks cleaner than string concatenation. e.g. 
        var book = "Harry Potter"
        var author = "J K Rowling"

        var FaveBook = `My favourite book is ${book} by ${author}`

            console.log(FaveBook);
            My favourite book is Harry Potter by J K Rowling.
We have used the same example as above and as you can see it is a lot easier to read.
